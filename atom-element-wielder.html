Atom Element Wielder

A human interface for interacting with the web, one element at a time.

Background/ Challenge/ Brief

Web components are building blocks of the web. Current documentation systems only allow users to view their API documentation or see some static examples. Before a component can be tweaked, it must be installed on oneâ€™s machine - an activity that relies on developer skills.

For any web component, the atom element wielder automatically generates a form using which the user can interact with the public API of the component - properties, methods and events.

Web components can be with or without a UI. The element wielder supports both cases. An interactive demo is generated for any given web component. In case of elements having a UI, that UI is shown in the demo. For data only components a demo is generated by the system.

For theming, users can modify the variables exposed by individual components, like how they modify properties - via a form that resembles most modern design tools. Additionally, a set of theme variables can be defined at the component set level, so those entire libraries of components can be themed in one go.

Primary Users
Designers, Business Analysts

Key Goals Addressed
Interact with freely available building blocks or those being prepared by their dev teams, to know what they can do and combinations in which they can be used - without coding.
